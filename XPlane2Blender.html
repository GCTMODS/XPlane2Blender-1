XPlane2Blender

Scenery & Cockpit import/export scripts for Blender

Blenderis an open source 3D object editor for Windows, Mac and UNIX. These Blender scripts export scenery created in Blender to X-Plane v7 or v8 .obj format for placement with World-Maker. The scripts also import existing X-Plane v6, v7 and v8 .obj scenery files and X-Plane v7 and v8 .acf airplanes into Blender. All X-Plane scenery primitives are supported apart from smoke_black and smoke_white and material attributes.

Requirements

Requires Windows 98 or later, Mac OS X 10.3 or later, or some version(?) of Linux.

Requires Blender 2.34 or later. Requires Blender 2.40 or later to export objects using animation.

Close Blender if you have it running.

 Extract the contents of the zip file to a new (temporary) folder.
Open the new folder.
Double-click on the file install.cmd.
(This installs the scripts in the folder %HOME%\blender\scripts if it exists, otherwise in the copy of Blender that is associated with .blend files).
Windows

(or wherever you installed Blender).

(This installs the scripts in the folder ~/.blender/scripts if it exists, otherwise in all copies of Blender that the Finder knows about).

Extract the contents of the zip file to this folder.

(Re)start Blender.This file can be viewed by choosing Help -> X-Plane in Blender.

You can now delete the temporary folder.

Importing objects

File -> Import -> X-Plane Object

Select a .obj file. The scenery is imported at the "3d Cursor" position. Any problems with the import are reported in the Console window as lines starting "Warn" or "Error".

The .obj file format doesn't preserve any information about which primitives belong together in the same mesh. So the script works quite hard to glue faces together into meshes for ease and speed of editing in Blender. This means that it can be very slow to import large scenery. You may find it helpful to manually merge some meshes together after importing; in "Object Mode" select the meshes you want to merge and choose Space -> Object -> Join Objects.

You can import multiple scenery files and re-export them as a single file. But note that the X-Plane .obj scenery file format only supports the use of one texture bitmap, so you'll have to create a single larger texture bitmap file containg all required textures. Use UVs -> Double bitmap size from the UV/Image Editor window to fix up texture mappings.

Importing planes

File -> Import -> X-Plane plane as scenery

Select a .acf file. The plane is imported with its reference point at the "3d Cursor" position. Any problems with the import will be reported in the Console window as lines starting "Warn" or "Error".

The script creates three versions of the plane in Blender, one in each of layers 1, 2 and 3, at three levels of detail. See Level of Detail below.

The imported plane needs some tweaking before you can export it as scenery; see Tweaking planes below.

Exporting objects

File -> Export -> X-Plane v7 Object or
File -> Export -> X-Plane v8 Object

The output filename and location is the same as the current Blender filename, but with a .obj extension. Always check the Console window for messages reporting any problems with the export, which are displayed as lines starting "Warn" or "Error".

v8: Note that v8 objects are only supported by X-Plane versions 8.20 and later.

Using Blender to create X-Plane objects

Lamps

  • R,G,B settings are exported to X-Plane. All other settings (including "Energy") are ignored.
  • Only Lamps of type "Lamp" are exported to X-Plane. Lamps of types "Area", "Spot", "Semi" and "Hemi" are ignored.
  • Lamps with certain words in their names have special behaviours when exported to X-Plane:
    Flash - Light flashes.
    Pulse - Red pulsing light. (R,G,B settings in Blender are ignored).
    Strobe - White strobe light. (R,G,B settings in Blender are ignored).
    Traffic - Cycles red, orange, green. (R,G,B settings in Blender are ignored).

Meshes

  • Create faces with 3 or 4 edges (called "tri"s and "quad"s in X-Plane).
  • In the Link and Materials panel:
    • Set Smooth and Set Solid control whether to smooth edges of faces in a mesh. This is useful when using multiple faces to simulate a curved surface. Go to "Object Mode", select the mesh and press Set Smooth. The effect is only visible in Blender 3D View windows when the Viewport Shading button is set to Solid or Shaded.
    v7: Only faces that are part of a Strip (see Strips below) will be smoothed when displayed in X-Plane.
  • In the Texture Face panel (available in UV Face Select mode):
    • Tex button controls whether the face has a texture. Create a .bmpor .png file in an image editor and save it in the custom object textures folder. Height and width must be powers of two (eg 256, 512, etc) but the bitmap doesn't have to be square. Draw all the textures required by the scenery onto it. Areas of the texture that are coloured magenta or that have Alpha transparency (png files only) will be transparent when displayed in X-Plane. Use the UV Image Editor window to control mapping of the texture bitmap to the face.
    • Tiles button controls whether the face is rendered with "polygon offsetting" (poly_os) in X-Plane. Use this for faces that lie horizontally on the ground to prevent artifacts in X-Plane.Don't press this button for other faces.
    v7: This button is ignored since "polygon offsetting" produces artifacts in X-Plane v7 and early versions of X-Plane v8.
    • Collision button indicates that the face is not "hard" (ie not "landable on") in X-Plane. Making faces hard is very expensive, so this button should normally be pressed. Unpress this button only for things like helicopter landing pads. (Note: The meaning of this button was changed in v1.50. Use this script to turn the Collision button back on for all faces).
      v7: Only faces with 4 edges (ie "quad"s) are exported as "hard".
      v7: X-Plane versions prior to 8.20 have a bug where .obj files that contain hard faces must be placed in WorldMaker with an "object heading" of 0. Otherwise the "hard" part of the surface ends up in the wrong place.
    • Twoside button controls whether one or both sides of the face are displayed. Using a two-sided face is more expensive than using a single-sided face, but is cheaper and easier than using two single-sided faces back-to-back.
    • Alpha button is a hint to Blender and to X-Plane that the face is transparent. Use this for transparent faces to prevent artifacts when displayed in X-Plane. Don't press this button for opaque (normal) faces.

The export script will change the order of faces in the output file to optimise for rendering speed and to ensure that faces with alpha are tendered after opaque faces. It will preserve the relative order of faces that have the same Link and Materials andTexture Face settings as each other.

Use this script if you need to reset all faces in the scene to standard settings (ie smooth, no polygon offsetting, not hard, single sided, not transparent).

Lines

Blender doesn't support Lines directly. Use a mesh with one 4-edged face instead. The pair of vertices at each end of the "line" must be within 0.1 units of each other. The face must be the only face in its mesh and must not have a texture assigned to it.

To colour the "line", assign a Material to the face and set the Material's Basic Colour RGB values. Faces not linked to a Material will be exported coloured grey.

v8: Animation

You can make lamps, meshes and lines animate in X-Plane according to the value of any of the "int" or "float" internal simulator datarefs listed here. In practice, the useful datarefs mostly have units of "bool" or "ratio".

Create an "Armature" object, with one or more "Bones". Name the bone(s) with one of the datarefs in the list. For datarefs that are shown as float[n] in the list, append [0] to the name to use the first value, [1] to use the second value etc. eg name the bone ENGN_thro[0] to represent the throttle setting for the first engine. Add additional bones to the armature and create parent/child relationships between each bone (in Edit Mode) if you need to use multiple datarefs to describe the animation of a single lamp or mesh.

Parent the lamp or mesh that you want to animate to the bone (choose the "youngest" child bone in the chain if you're using multiple datarefs):

  • Click on the lamp or mesh
  • Shift-click on the Armature
  • Choose Pose Mode from the Mode menu
  • Click on the bone
  • Press Ctrl-P and select Bone from the popup menu

Use Pose Mode to describe the animation; insert keys in Blender's frames 1 and 2 to represent the desired position of the mesh at the minimum and maximum dataref values respectively. (X-Plane's animation syntax is quite simple; so don't use IPOcurves or any other advanced Blender animation techniques since these will be ignored by the exporter - only the positions in frames 1 and 2 are exported to X-Plane).

By default, the minimum and maximum dataref values are 0 and 1 respectively. You can change the values by adding properties in the Logic panel (F4); name the property after the dataref and append _v1 to the name for the minimum value or _v2 for the maximum value. eg the yoke pitch dataref yolk_pitch_ratio has values between -1 (forward) and +1 (back) in X-Plane; so to get X-Plane to use the correct minimum dataref value for the animation add a property called yolk_pitch_ratio_v1 and give it the value -1.

Creating X-Plane scenery

  • Find and open the Custom Scenery folder inside your X-Plane or X-System nnn installation (where nnn is the version number of your version of X-Plane - eg X-System 763).
  • Create a subfolder with the name of the scenery collection that you're making.
  • Open the new folder.
  • Create two new subfolders, called custom objects and custom object textures (these names must be spelt correctly).
  • Save your Blender file in the custom objects folder with a descriptive name, eg:
    X-System nnn\Custom Scenery\EGLL\custom objects\house.blend
    The X-Plane .obj file will be exported to the same place, ie:
    X-System nnn\Custom Scenery\EGLL\custom objects\house.obj
  • Use Lamps and Meshes to construct your scenery. No other Blender object types are exported.

Tweaking planes

Imported planes need some tweaking to be positioned correctly on the ground:

  • Select layers 1-3, select all objects and position the plane so that its undercarriage is sitting directly on the ground (represented by the x/y axes).
  • You may also need to rotate the plane slightly so that all wheels are level; press r and move the mouse to rotate the plane so that its undercarriage is sitting directly on the ground. Click to set the plane's position.

The main texture bitmap file is named airplane_paint. Most planes also use textures from a second bitmap file named airplane_paint2. If your imported plane uses weapons then each weapon will also use an additional bitmap file. However the X-Plane .obj scenery file formats only support the use of a single texture bitmap file. Meshes that use textures from an additional bitmap file are imported with "*" after their name.

Open an Outliner window, and for each listed mesh that has a name ending with "*", either:

  • Delete the mesh,
  • Or copy the required textures to an unused area in airplane_paint, and use the UV Image Editor window to map the new textures in airplane_paint to the mesh's faces.

Consider performance issues when the plane is rendered in X-Plane. Ask yourself the following questions:

  • Most important: Do you really need fully detailed 1024x1024 textures for your static plane? Video memory is used up by terrain and object textures. Once you run out of video memory the GPU has to fall back to main memory, and this really slows things down. One 1024x1024 texture uses 4MB of video memory. Some people are running X-Plane on computers with only 32MB of video memory, so one 1024x1024 texture at highest resolution uses 1/8th of it. Consider resizing the texture bitmap file in an image editor to 512x512 or even 256x256. Use Image -> Replace to use the new texture bitmap file.
  • Does the model have hidden faces? Some versions of X-Plane don't handle hidden faces in v7 scenery objects very well and they also a cause a small performance hit. Look for things like wings or misc bodies that are partially or wholly buried in the fuselage and delete any wholly hidden faces before exporting.
  • Do you really need all that detail? Consider deleting details like flap tracks, antennae etc before exporting. This especially goes for the lower Level of Detail versions of the plane in layers 2 and 3 which are only viewed in X-Plane from >1000m and >4000m respectively.

Creating 3D cockpits

X-Plane 3D cockpits are just normal v7 or v8 scenery objects. However cockpits can't contain multiple Levels of Detail, so only objects in Blender layer 1 are exported. If you want to keep objects in your Blender scene for reference but which you don't want to export - eg the plane fuselage - then put them in layer 4 or greater before exporting.

Choose File->Save As and save the blender file in the same folder as your plane's .acf file with the name airplane_cockpit.blend, where airplane is the name of your plane's .acf file.

Normally you will want to import your plane into Blender to act as a reference and/or starting point for your cockpit. Delete any plane parts that you don't need in creating your cockpit - you only need to keep the fuselage itself plus any relevant Misc Bodies. After import, your cockpit uses the same texture file as your plane, ie airplane_paint. Choose Image -> Replace in a UV Image Editor window to use a different texture file, which can be named anything you like but which should live in the same folder as your plane's .acf file.

To construct moving cockpit instruments in X-Plane v8:

  • Layout the 2D panel in PlaneMaker as normal. But bear in mind that only the top 768 lines of the 2D panel can be used in the 3D cockpit in X-Plane versions prior to 8.20. This is fixed in X-Plane v8.20.
  • Run X-Plane and load your aircraft. Press the Up arrow to ensure that you're looking at the top of the 2D panel. Take a screenshot (on a PC press Alt PrtSc).
  • Load the screenshot into an image editor and change its size so that it's the same size as your 2D panel (typically 1024x1024) - don't resize the screenshot, just add blank space. Save the screenshot as ScreenshotPanel.png (or any filename ending in *panel.*) in the same folder as your plane's .acf file.
  • Use the ScreenshotPanel.pngtexture on those faces that you want to display moving cockpit instruments in X-Plane. (It's easier to use this screenshot texture instead of the real panel texture while editing in Blender, since the screenshot has instruments drawn on it). The screenshot file does not need to be distributed with your finished plane.

Note that X-Plane versions prior to 8.20 only display the 3D cockpit when running at the default 1024x768 resolution. You may want to mention this in the Readme with your plane if your plane works with X-Plane versions prior to 8.20. This is fixed in X-Plane v8.20.

Optimising for X-Plane

v7: Strips

As well as basic 3- and 4-edged faces ("tri"s and "quad"s), X-Plane v7 objects support two compound types - "tri_fan" and "quad_strip". These are strips of two or more tris and quads that share common edges. Because the faces in these strips share common edges, X-Plane and the underlying OpenGL renderer have a third or a half as much work to do to render them compared to individual tris and quads. This gives higher frame rates.

In order for a pair of faces to be considered for inclusion in one of these strips, the following conditions need to be true:

  1. The faces must be facing the same way.
  2. Each pair of faces must share a common edge (apart from the first and last face).
  3. Each shared edge must have the same texture bitmap co-ordinates in both faces.

In practice this means that the texture must be reversed in alternate faces in the strip. In the case of tris this can also be achieved by mapping a single area of the texture across all the tris, with the tip of the tris at the centre of the texture area. Use UVs -> Copy & Paste from the UV/Image Editor window to automate the creation of strips.

These compound types aren't supported directly by Blender. However, the export script automatically tries to spot when it can use them.

v8: These compound types aren't supported in v8 objects. X-Plane uses more efficient techniques in v8 objects.

Level Of Detail

X-Plane has borrowed the concept of "Level Of Detail" from 3D games. This works on the principle that objects can be drawn with less detail when you're viewing them from a large distance.

Use Layers to draw scenery with multiple Levels Of Detail. Objects in layers 1-3 are visible in X-Plane at the following distances:

Layer Distance
1 < 1000m
2 1000-4000m
3 > 4000m

Blender tips

Blender has a quirky user interface:

  • By default, Right mouse button does what you would expect Left mouse button to do. I strongly recommend that you change this; drag down the top menubar to reveal the User Preferences window, select the View & Controls tab and choose Select with: Left Mouse. See this tutorial for a fuller explanation.
  • Space does what you might expect Right mouse button to do.
  • The user interface is "modal". Buttons and keys do different things depending on whether you are in "Object", "Edit" or "UV Face Select" mode. (And I thought modal UIs went out with "vi"). Tab changes mode.

Limitations

Import object

  • Animation is not supported.
  • smoke_black and smoke_white X-Plane primitives are ignored.
  • poly_os, ambient, difuse, specular, emission and shiny attributes are ignored.
  • The importer usually can't work out which faces are partially or wholly transparent. You should tell Blender which faces are transparent faces by pressing the Alpha button in UV Face Select mode after import.

Import plane

  • Importing v8.20 planes is not yet supported. v7.40 to v8.17 planes should work fine.
  • Cockpits objects are ignored. You can use File -> Import -> X-Plane Object to import cockpits.
  • Planes (and their weapons) can use multiple texture bitmap files. These are not automagically merged into one file during the import.
  • The importer can't work out which faces are partially or wholly transparent. You should tell Blender which faces are transparent faces by pressing the Alpha button in UV Face Select mode after import.
  • LOD algorithm can be too conservative in layer 2.
  • v6 planes are not supported. Convert a v6 plane to v7 by opening it in PlaneMaker 7.40 or later and re-saving it.

Export object

  • Only Lamps and Mesh Faces (including "lines") are exported.
  • All faces must share a single texture bitmap file (this is a limitation of the X-Plane .obj file formats) apart from cockpit panel faces which can additionally use the cockpit panel bitmap file. Multiple textures are not automagically merged into one file during the export.
